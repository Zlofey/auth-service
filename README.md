# auth-service

Сервис аутентификации с JWT токенами, refresh token rotation и защитой от race conditions.

## Ключевые архитектурные решения

- **Независимость Content Service:** Сервис контента не обращается к Redis и не валидирует токены. Это устраняет лишние сетевые вызовы, упрощает масштабирование и снижает связанность компонентов.
- **Отказоустойчивость при падении Auth Service:** Если Auth Service становится недоступен, пользователи сохраняют доступ к контенту до истечения срока действия их Access Token. Gateway продолжает работать, проверяя локальную подпись и статус в Redis.
- **Слабая связанность:** Компоненты общаются через стандартизированные заголовки и общие контракты данных. Gateway выступает единственным посредником, скрывая внутреннюю структуру системы от внешнего клиента.
- **Черный список в Redis:** Реализован черный список в Redis, что позволяет проверять отозванные токены, не нагружая БД и даже не трогая Auth Service.
- **Белый список:** Реализован белый список (таблица сессий в Postgres, поле `refresh_jti`), что позволяет, например, отозвать все сессии пользователя.

## Возможности

- Регистрация, смена пароля, логин, логаут, логаут из всех устройств
- Refresh token rotation с защитой от race conditions
- Blacklist отозванных токенов в Redis
- Поддержка ролей: admin, staff, client
- Логирование запросов (ip, user_agent)


## Запуск через Docker Compose

```bash
# Копируем .env.example в .env (если нужно настроить переменные)
cp .env.example .env

# Запускаем все сервисы
docker compose up -d

# Останавливаем сервисы
docker compose down

# Смотрим логи
docker compose logs -f app
```

Сервис доступен по адресу (по умолчанию):

- API документация: http://localhost:8000/docs

При запуске контейнера автоматически применяются все миграции базы данных через Alembic.

## API эндпоинты

- `POST /auth/register` - регистрация нового пользователя
- `POST /auth/login` - логин
- `POST /auth/refresh` - обновление токена
- `POST /auth/logout` - логаут
- `POST /auth/logout-all` - логаут из всех устройств
- `POST /change-password` - изменение пароля пользователя
